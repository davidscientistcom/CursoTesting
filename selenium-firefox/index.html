Qué es automatizar pruebas

Automatizar pruebas significa escribir código que abre una aplicación, reproduce acciones humanas y comprueba resultados observables. La clave no es “hacer clics automáticamente”, sino convertir comportamientos esperados del sistema en verificaciones repetibles.

Ventajas

  • Repites un flujo cien veces sin depender del cansancio o de la memoria humana.
  • Detectas regresiones cada vez que cambias una pantalla, un flujo o una validación.
  • Documentas el comportamiento esperado con código ejecutable.
  • Ganas velocidad en regresión, smoke tests y validaciones repetitivas.

Desventajas y límites

  • No todo merece automatización: casos raros o flujos muy volátiles pueden salir caros.
  • Si el DOM cambia y tus selectores son malos, los tests fallan por la capa equivocada.
  • Si no sincronizas bien, verás falsos negativos por tiempos de carga.
  • Una suite mal diseñada también cuesta mantenerla.
WARN
Regla docente de este curso: trabajamos contra una web controlada porque usar Facebook, Google o cualquier SaaS público te mete ruido que aquí no aporta nada. Cambian ids, textos, políticas anti-bot y flujos. Para aprender Selenium, eso es un estorbo.

Qué es Selenium y cómo ha evolucionado

Selenium es el ecosistema clásico de automatización de navegadores. Nació con herramientas distintas para necesidades distintas y con el tiempo convergió alrededor de WebDriver como estándar. Hoy Selenium 4 trabaja sobre el protocolo W3C WebDriver y añade Selenium Manager para facilitar la gestión de drivers.

Pieza Qué hace Cuándo usarla

WebDriver API programable para controlar el navegador Automatización seria, mantenible y versionable
Selenium Grid Ejecución distribuida en varios nodos y navegadores Cuando una sola máquina ya no escala

Instalación paso a paso con Conda 3.13 y Firefox

A fecha de 9 de abril de 2026, el camino simple para este curso es: crear un entorno Conda con Python 3.13, instalar Selenium con pip, comprobar Firefox y levantar la web de prácticas con un servidor HTTP estándar. No hace falta montar backend ni base de datos.

bash - Preparación completa del entorno 
conda create -n selenium-firefox-313 python=3.13 pip -y
conda activate selenium-firefox-313
python -m pip install --upgrade pip selenium

firefox --version
geckodriver --version

cd lucentia/output/learning/selenium-firefox
python -m http.server 8765
INFO
Qué consigues con esto: el curso queda servido en http://127.0.0.1:8765/ y el laboratorio en http://127.0.0.1:8765/lab/. Los ejemplos Python asumen esa URL base.
bash - Validar que todo arranca antes de seguir 
conda activate selenium-firefox-313
cd lucentia/output/learning/selenium-firefox
python examples/validate_all.py
WARN
Si Firefox no arranca: primero revisa el binario y el driver, no el test. Un fallo de arranque del navegador no significa que tu localizador esté mal; significa que el entorno todavía no está preparado.
