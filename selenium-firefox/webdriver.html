Localizadores: del más estable al más frágil

En un curso real, enseñar Selenium sin enseñar criterios de localización es dejar al alumno vendido. El localizador no es un detalle: es la bisagra entre tu test y el DOM.

Estrategia Cuándo preferirla Riesgo
id Cuando la página define ids estables y únicos Bajo
name Formularios clásicos Bajo si el nombre es semántico
CSS selector Cuando necesitas combinar clase, atributo y jerarquía sencilla Medio si te pasas de específico
XPath Cuando CSS no basta o necesitas texto visible Sube rápido si te apoyas en estructura frágil
INFO
Criterio práctico: primero intenta id, después name, luego un CSS simple. XPath es útil, pero no debe ser tu primer refugio para todo.

Métodos básicos y excepciones que debes conocer ya

Métodos básicos

  • get() abre una URL.
  • find_element() y find_elements() localizan nodos del DOM.
  • send_keys() escribe texto.
  • click() dispara acciones de usuario.
  • current_url y title te permiten validar navegación.

Excepciones base

  • NoSuchElementException: el elemento no existe cuando lo buscas.
  • TimeoutException: la condición esperada no se cumple a tiempo.
  • Además de capturas específicas, conviene tener un except Exception as exc genérico como último paraguas.
FAIL
Error muy común: lanzar una excepción y dejar Firefox abierto porque el script termina a mitad. La forma seria de evitarlo es un finally que cierre el navegador siempre.
