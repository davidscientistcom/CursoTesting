Capítulo 1

Arranque limpio del laboratorio

Aquí no hay pasos implícitos. El laboratorio está preparado para reconstruirse desde cero. Lo importante no es solo levantar contenedores, sino entender qué pieza resuelve cada problema operativo.

Paso 1. Entrar en el directorio del laboratorio

Shell
cd /home/deivit/Lucentia/projects/jenkins-lab

Todo el material necesario está ahí: `docker-compose.yml`, scripts, plantilla del repo y documentación HTML.

Paso 2. Levantar y verificar el entorno completo

Shell
./scripts/run-end-to-end-check.sh

Este script no solo levanta contenedores. También crea el usuario de Gitea, genera el repositorio remoto, configura el webhook y ejecuta la demo completa del fallo y la reparación.

Qué hace cada servicio

Servicio Papel técnico Por qué existe
Gitea Remoto Git local El alumno empuja código a un servidor real de Git, no a un directorio simulado.
Jenkins Orquestador CI Lee el `Jenkinsfile`, clona el repo y ejecuta stages separados para unitarios e integración.
Webhook relay Puente HTTP interno Gitea hace `POST` al relay y el relay dispara el build de Jenkins. Se evita así depender del comportamiento de polling del plugin Git.
