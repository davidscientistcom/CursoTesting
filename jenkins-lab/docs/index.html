Jenkins LabCI reproducible con Gitea, pytest y MariaDB
0. Índice 1. Arranque 2. Gitea + Jenkins 3. Código + unitarios 4. Integración + fallo 5. Reparación
Laboratorio verificado

Un flujo de CI que no se explica en abstracto: se levanta, se rompe y se repara.

Este laboratorio enseña un pipeline de Jenkins sobre código Python real. El alumno ve un push en Gitea, observa cómo Jenkins ejecuta test unitarios y de integración contra MariaDB, provoca un fallo en el adaptador SQL y comprueba después la reparación en un build verde.

Objetivo didáctico

Entender la cadena completa: repositorio remoto, job de Jenkins, stages con contenedores, diferencia entre mocks y base de datos real, y lectura de fallos reales en consola.

Servicios del stack

El stack levanta tres piezas: Gitea, Jenkins y un relay HTTP interno. El relay existe para que el push dispare el job de Jenkins de manera determinista sin depender del polling del plugin Git.

Resultado verificado

La secuencia real quedó validada con dos builds consecutivos: el build 3 falla por el bug SQL y el build 4 pasa tras corregir el filtro de consulta.
