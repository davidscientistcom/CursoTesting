Capítulo 3

La aplicación de ejemplo y el sentido de los test unitarios

El código del laboratorio no es un “hola mundo”. Hay una clase de dominio, un contrato de persistencia y un servicio que depende de ese contrato. Esa separación es la base para entender por qué los unitarios pueden pasar aunque el acceso real a la base esté roto.

Diseño mínimo del ejemplo

Estructura Python
src/gradebook/
├── domain.py              # GradeRecord
├── repository.py          # Protocolo GradeRepository
├── service.py             # GradebookService
└── mariadb_repository.py  # Adaptador real a MariaDB

GradebookService aplica reglas simples: valida entradas, registra notas y calcula la media del alumno. La persistencia queda fuera, detrás de un contrato, para que la lógica se pueda probar sin depender de red, base de datos o Docker.

Qué prueban los unitarios

  • Que se persiste el registro correcto.
  • Que la media se calcula solo con las notas del alumno pedido.
  • Que si no hay datos, la media es 0.0.
  • Que las notas fuera de rango disparan ValueError.

Qué no prueban los unitarios

  • No prueban SQL.
  • No prueban el nombre de columnas.
  • No prueban conectividad con MariaDB.
  • No prueban si una consulta devuelve o no filas reales.
Idea clave para clase. Un test unitario en verde no significa que el sistema completo esté sano. Solo significa que la unidad aislada cumple su contrato bajo el doble de prueba que le hemos dado.

Stage unitario del Jenkinsfile

Jenkinsfile
stage('Tests unitarios') {
    steps {
        script {
            docker.image('python:3.12-slim').inside('-u root:root') {
                sh '''
                    mkdir -p reports
                    python -m venv .venv
                    . .venv/bin/activate
                    pip install -e .[test]
                    pytest tests/unit --junitxml=reports/unit.xml
                '''
            }
        }
    }
}

Este stage ya enseña dos mensajes importantes: Jenkins ejecuta el trabajo dentro de un contenedor controlado y publica el XML JUnit para que el alumno pueda inspeccionar el resultado en la interfaz.
