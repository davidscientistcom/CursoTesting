Python Idiomático

Ser "Pythonic" es una de las cosas más hermosas del lenguaje. El libro original de Clean Code fue escrito en Java, pero Python tiene características nativas que reemplazan patrones clásicos pesados con construcciones declarativas limpias.

1. Comprensiones y Generadores (List / Dict Comprehension)

En lugar de crear una lista vacía y hacer un bucle for que utilice .append() iterativamente, Python ofrece comprensiones de listas, sets y diccionarios, que son más veloces de evaluar y más legibles.

❌ Mala Práctica (For-Loop verboso)
def get_active_user_emails(users):
    emails = []
    for user in users:
        if user.is_active:
            emails.append(user.email)
    return emails
✅ Buena Práctica (Comprehension)
def get_active_user_emails(users):
    return [
        user.email 
        for user in users 
        if user.is_active
    ]

# Si son muchos, usa un generador `( ... )`
# en vez de corchetes de lista `[ ... ]`

2. Context Managers (El estatuto `with`)

Nunca gestiones manualmente la apertura y cierre de recursos (archivos, conexiones a BD, locks). En lugar de bloques try-finally pesados, utiliza Context Managers.

❌ Manual y Propenso a errores
def write_log(msg):
    f = open('app.log', 'a')
    try:
        f.write(msg + '\n')
    finally:
        f.close()
✅ Context Manager Seguro
def write_log(msg):
    with open('app.log', 'a') as f:
        f.write(f"{msg}\n")
    # Python cierra el archivo 
    # automáticamente aquí

3. Desempaquetado y Funciones Pythonic (`enumerate`, `zip`)

Si alguna vez encuentras tu código usando range(len(lista)) solo para acceder por índice lista[i], detente. Usa iterables directamente. Y si necesitas el índice, usa enumerate.

❌ Iteración C-Style
names = ["Alice", "Bob", "Charlie"]

for i in range(len(names)):
    print(i, names[i])

# Swap verboso
temp = a
a = b
b = temp
✅ Iteración Pythonic
names = ["Alice", "Bob", "Charlie"]

for idx, name in enumerate(names):
    print(idx, name)

# Swap express
a, b = b, a

4. Type Hints: La Mejor Documentación

Python 3 introdujo y maduró el uso de Type Hints (Anotación de Tipos). Al utilizar tipado, pasas la validación a tu IDE y a herramientas como `mypy`. Tu código se vuelve autodescriptivo sin necesitar Docstrings para indicar de qué tipo son los parámetros y qué devuelves.

❌ Tipado en Comentarios
def get_user(id_str):
    """
    Agara un usuario
    :param id_str: str con uuid
    :return: dict o None
    """
    pass
✅ Type Hints Nativos
from typing import Optional

def get_user(id_str: str) -> Optional[dict]:
    # El IDE y Mypy entienden 
    # la firma de la función.
    pass
