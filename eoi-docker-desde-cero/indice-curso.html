Imagenes, contenedores, redes, volumenes y Compose usando un laboratorio real

Este curso sustituye a la version anterior. El hilo conductor ya no es un texto abstracto, sino el laboratorio real en /home/deivit/Lucentia/projects/jenkins-lab. Durante esta sesion se ha vuelto a ejecutar ./scripts/run-end-to-end-check.sh: se reconstruyeron las imagenes, se levantaron los tres servicios, se creo el repositorio en Gitea, un primer push dejo un build en rojo y un segundo push dejo el pipeline en verde. Que se ha comprobado de verdad

No es una promesa del material. Es lo que ha ocurrido al ejecutar el laboratorio.

IMG

Build reproducible de imagenes

Se han reconstruido las imagenes jenkins-lab-jenkins y jenkins-lab-webhook-relay con docker compose up -d --build.

RUN

Servicios levantados

docker compose ps mostro gitea, jenkins y webhook-relay en ejecucion, con Jenkins marcado como healthy.

VOL

Persistencia declarada

docker compose config --volumes devolvio gitea_data y jenkins_data, que son la base de la persistencia del laboratorio.

LAB

Flujo completo de clase

El script integral creo el repo, lanzo un push con bug y confirmo un segundo push con reparacion. En esta sesion quedaron verificados un build fallido y un build exitoso.
