Decoradores y fixtures — referencia

Decoradores de test

@pytest.mark.skip(reason="...") — siempre salta
@pytest.mark.skipif(cond, reason=) — salta si condición
@pytest.mark.xfail(reason=) — fallo esperado
@pytest.mark.parametrize("x,y", [...]) — múltiples casos
@pytest.mark.timeout(5) — mata si tarda >5s
@pytest.mark.asyncio — test async (pytest-asyncio)
@freeze_time("2024-01-15") — congelar tiempo (freezegun)

Fixtures — opciones

@pytest.fixture — scope="function" (una por test)
@pytest.fixture(scope="module") — una por fichero
@pytest.fixture(scope="session") — una por ejecución
@pytest.fixture(autouse=True) — aplica sin nombre
yield — código antes=setup, después=teardown
request.param — acceder al parámetro en fixture
@pytest.fixture(params=[...]) — fixture parametrizada

Assertions especiales

pytest.raises(ExcType) — verificar excepción
pytest.raises(ExcType, match="...") — + mensaje
exc_info.value.atributo — acceder a la excepción
pytest.approx(0.1 + 0.2) — float con tolerancia
assert x is None — identidad (no == None)
assert x is True/False — identidad booleana
assert x in coleccion — pertenencia

conftest.py — dónde va cada cosa

tests/conftest.py → fixtures de toda la suite
tests/unit/conftest.py → solo para tests unitarios
tests/integration/conftest.py → solo para integración

No se importan: pytest los descubre automáticamente.
Las fixtures de un conftest padre son accesibles
desde todos los subdirectorios.

Método / atributoDescripción
MagicMock(spec=ClaseReal)Mock que respeta la interfaz de ClaseReal

mock.metodo.side_effect = excLanza la excepción cuando se llama

Hypothesis — strategies de referencia

Tipos básicos

st.integers()
st.integers(min_value=0, max_value=100)
st.floats(allow_nan=False, allow_infinity=False)
st.decimals(min_value="0", max_value="100", places=2)
st.text()
st.text(min_size=1, max_size=50)
st.binary()
st.booleans()
st.none()
st.emails()
st.from_regex(r"\d{4}-\d{2}-\d{2}")

Colecciones

st.lists(st.integers())
st.lists(st.integers(), min_size=1, max_size=10)
st.sets(st.integers())
st.frozensets(st.integers())
st.tuples(st.integers(), st.text())
st.dictionaries(st.text(), st.integers())
st.one_of(st.integers(), st.text())
st.integers() | st.text() # equivalente a one_of
st.just(42) # siempre el mismo valor
st.sampled_from([1, 2, 3]) # uno de la lista

Objetos y builds

from hypothesis.strategies import builds

builds(MiClase, campo=st.integers())

# st.from_type usa las anotaciones de tipo
st.from_type(MiDataclass)

# Composite: combinar estrategias
@st.composite
def mi_strategy(draw):
  n = draw(st.integers(min_value=1))
  return [draw(st.integers())] * n

SOLID — resumen ejecutivo

PrincipioEn una líneaSeñal de violación en tests
S — SRP Una clase, una razón para cambiar Necesitas levantar infraestructura para testear lógica pura
O — OCP Abierto a extensión, cerrado a modificación Añadir un caso nuevo rompe tests existentes
L — LSP Los subtipos son sustituibles por el tipo base La misma suite de tests falla para algunas implementaciones del Protocol
I — ISP Interfaces pequeñas y focalizadas El Mock necesita implementar métodos que el test no usa
D — DIP Depender de abstracciones, no de detalles No puedes inyectar un mock — la dependencia está hard-coded

Checklist de calidad por tipo

Integers / floats — qué testear siempre

Valor nominal (caso típico)
Cero
Negativo (si aplica / si debe fallar)
Mínimo del dominio
Máximo del dominio
Valor en el límite exacto (±1 del umbral)
None (si el tipo es Optional)
Precisión flotante (usar pytest.approx)

Strings — qué testear siempre

Valor nominal
Cadena vacía ""
Solo espacios " "
Espacios al inicio y fin " valor "
Mayúsculas / minúsculas
Caracteres unicode
Longitud mínima y máxima
None (si Optional)

Listas — qué testear siempre

Lista vacía []
Un solo elemento
Elemento duplicado
Lista desordenada (si el orden importa)
Lista ya ordenada
Todos los elementos iguales
Lista muy larga (Hypothesis)
None (si Optional)

Excepciones — qué verificar siempre

Tipo exacto (no solo Exception)
Mensaje con match="texto clave"
Atributos de la excepción si los tiene
Estado del objeto NO cambió tras el error
No se lanza en el caso válido
Se lanza con el input mínimo incorrecto

  ─────────────────────────────────────────────────────────────

  ruff check        pre-commit        <3s      Commit local

    pip install pytest pytest-cov pytest-mock ruff mypy bandit pre-commit
