¿Por qué ruff?

Ruff es mucho más que un linter: combina el chequeo de estilo, la detección de errores comunes y un formateador ultra rápido. En lugar de encadenar flake8, black, isort e incluso algunos plugins de pylint, con ruff tienes todo en un solo binario que funciona en segundos. Es capaz de corregir el código automáticamente con ruff format, encontrar importaciones sin usar, variables no usadas, nombres de funciones incorrectos, errores de seguridad simples y más (mira la salida en la pestaña de Demos). 

# ejemplo rápido de uso:
ruff check src/bad_practices_showcase.py  # muestra advertencias
ruff format src/bad_practices_showcase.py --diff  # aplica formato y muestra cambios

Si vienes de flake8/black, la curva de aprendizaje es mínima y la velocidad hace que los hooks pre-commit terminen casi instantáneo.

Novedades/ajustes que suelen faltar

Ruff como estándar

Muchos equipos han sustituido combinaciones antiguas (flake8 + isort + black) por ruff para simplificar y acelerar.

Tendencia fuerte

Mutación selectiva

No para todo el repo. Úsala en módulos críticos (pricing, fraude, permisos) para validar la calidad real de tests.

Coste moderado

Contract testing

Con APIs/microservicios, pact-style testing reduce roturas entre consumidor y proveedor.

Muy útil en microservicios

Conclusión directa sobre tu lista

Si incluiste herramientas clásicas de testing Python, vas bien encaminado. Para dejarla redonda hoy:

  • Mantén pytest + coverage + pre-commit como núcleo.
  • Añade ruff y mypy/pyright como gates de calidad.
  • Introduce hypothesis y/o mutación en zonas críticas.
  • Incluye seguridad básica: pip-audit y bandit.
