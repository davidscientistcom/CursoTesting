Ejecuta estas herramientas sobre un archivo de muestra

Copia y pega los comandos en tu terminal dentro de /home/deivit/curso_testing para ver la potencia de cada utilidad.

1. RUFF: lint, fix y velocidad

Analiza estilo, errores y puede formatear todo. Está reemplazando a flake8+black+isort en muchos equipos.

bash
# instala ruff si no lo tienes
pip install ruff

# revisa el fichero de malas prácticas
ruff check src/bad_practices_showcase.py

# repara automáticamente algunos problemas (formato, imports)
ruff format src/bad_practices_showcase.py

# vuelve a ejecutar para ver solo avisos restantes

Resultados esperados:

  • Importaciones sin usar, nombres en CamelCase, espacios extra, eval/shell, etc.
  • Al ejecutar ruff format, el archivo se reescribe con estilo correcto.

2. mypy / pyright: chequeo de tipos estático

Detecta que Calcula_Suma se declara con parámetros str pero se llama con enteros.

bash
pip install mypy
mypy src/bad_practices_showcase.py

# o usando pyright (instala con npm)
# npm install -g pyright
# pyright bad_practices_showcase.py

El informe mostrará incompatibilidades de tipos y nos recuerda declarar tipos correctos.

3. pytest con cobertura

Ejecuta todos los tests (unittest/pytest se mezclan) e imprime un informe de cobertura.

bash
pip install pytest pytest-cov

# ejecuta tests y genera cobertura en terminal
pytest -q --disable-warnings --cov=src/bad_practices_showcase.py --cov-report=term-missing

# visualiza el HTML (si quieres)
pytest --cov=src/bad_practices_showcase.py --cov-report=html
xdg-open htmlcov/index.html

Verás líneas sin ejecutar (función funcion_sin_tests, print muerto, etc.).

4. hypothesis para encontrar errores ocultos

Corre el test property-based del archivo y deja que genere cientos de casos.

bash
pip install hypothesis
pytest tests/test_bad_practices_showcase.py::test_con_hypothesis -q

# verás un fallo inmediato; hypothesis imprime el caso minimizado

5. bandit + pip-audit para seguridad

Bandit buscará patrones inseguros en el código, pip-audit listará vulnerabilidades de dependencias.

bash
pip install bandit pip-audit
bandit -r src/bad_practices_showcase.py
pip-audit

Bandit marcará el hardcoded password y el uso de eval o shell=True.

6. coverage + mutmut (opcional)

Puede medir cuántas mutaciones tus tests detectan.

bash
pip install mutmut
# inicia mutación en el archivo; detén con Ctrl-C después de unos segundos
mutmut run src/bad_practices_showcase.py
mutmut results

Verás qué mutaciones (cambios sintéticos) pasan los tests - permite detectar tests débiles.
