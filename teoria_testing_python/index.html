Flujo completo
Idea central: no basta con “tener tests”; hay que combinar pruebas, estilo, tipado, seguridad y CI para reducir fallos de verdad.

Arquitectura recomendada del ciclo de calidad

flowchart LR A[Commit] --> B[pre-commit: ruff + format + mypy rápido] B --> C[pytest unit + mutation crítica] C --> D[coverage gates] D --> E[CI: integración + e2e + seguridad] E --> F[Deploy]
🎯 Cobertura útil: 80%-90% (no obsesión por 100%)
⚡ Feedback local: < 60s
